# Papaipay Blueprint v1

> **Legacy notice:** This document is retained for historical reference only. Current product direction is defined in `docs/product-blueprint-v2.md`. Product Blueprint V2 defines two products: Corporate Website and Asset Participation Platform, with Member and Admin as the primary authenticated roles.


## Project Overview

Papaipay is a Debt Advisory and Asset Management Platform designed to support members through financial assessment, document collection, advisory case management, consent management, asset matching, contribution tracking, and progress monitoring.

The MVP focuses on operational efficiency, transparency, structured advisory workflow, and member case management.

---

## User Roles

- View website
- Learn about Papaipay

- Register account
- Complete profile

- View contribution history

### Advisor

- View assigned members
- Review member profiles
- Review financial assessment
- Follow up members
- Add notes
- Update case progress
- Prepare advisory recommendations

### Admin

- Full system access
- Manage members
- Manage advisors
- Manage assets
- Review assessments
- Manage cases
- Manage contributions
- View reports
- Monitor activity logs

## MVP Objective

The primary objective of the MVP is to provide a working platform where:

- Members can register and submit financial assessments
- Members can upload supporting documents
- Members can provide consent for advisory review
- Advisors can monitor and manage assigned members
- Advisors can create notes and track advisory cases
- Admins can manage members, advisors, assets, cases, and contributions
- System activities can be tracked transparently

---

## Compliance-Aware Design Direction

The platform should support clear, traceable, and documented workflows.

The system should prepare for:

- Member consent records
- Advisory notes
- Case progress tracking
- Document records
- Assessment records
- Activity logs
- Communication records
- Reports and exports
